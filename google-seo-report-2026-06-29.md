## 六、问题与风险

### 🔴 高优先级
1. **GSC 自动化完全受阻**
   - 没有服务账号 API 密钥
   - 浏览器自动化需要 Google 登录
   - 无法获取点击量、展示量、平均排名、覆盖范围等核心指标

2. **无法验证收录健康度**
   - 自 6 月 2 日以来没有新的 GSC 真实数据
   - 无法判断新增 16 篇文章是否被收录
   - 无法检测收录量骤降或错误页面

### 🟡 中等优先级
3. **site: 搜索验证被 Google 拦截**
   - 自动化搜索触发反爬虫验证
   - 不能作为可靠的替代方案

4. **Sitemap 快速扩张**
   - URL 数量增长快，可能超出 Google 抓取预算
   - 需确保新增内容质量并主动提交

## 七、建议措施

### 立即行动（本周）
1. **手动登录 GSC 检查以下数据**：
   - 效果 → 总点击次数、展示次数、平均排名（最近 28 天）
   - 覆盖范围 → 已收录、未收录、错误页面数量
   - 体验 → 核心网页指标、移动设备易用性

2. **对新增的 16 篇博客文章手动请求编入索引**
   - GSC → URL 检查工具 → 逐个输入新文章 URL → 请求编入索引

### 中期解决（本月）
3. **配置 GSC API 服务账号**（一劳永逸）
   - 在 Google Cloud Console 创建项目
   - 启用 Google Search Console API
   - 创建服务账号并下载 JSON 密钥
   - 将密钥文件保存为：`.workbuddy/automations/google/gsc-service-account-key.json`
   - 在 GSC 中将服务账号邮箱添加为「完整权限」用户
   - 完成后，自动化脚本 `gsc_api_check.py` 即可直接读取 GSC 数据

### 长期优化
4. **考虑 URL 静态化**
   - 将 `blog-post.html?id=X` 改为 `/blog/X/` 或 `/blog/slug/`
   - 更利于 Google 索引和用户体验

5. **建立收录监控基线**
   - 设定每周收录量变化阈值
   - 配置异常自动告警
